# Changelog

## 0.4.0 — 2026-04-02

### Setup wizard
- Post-registration onboarding at `/setup` — 4-step stepper with API polling
- Step 1: Log in check (auto-completes if authenticated)
- Steps 2-3: Poll sites and workforces endpoints to confirm data flow from FIO extension
- Step 4: Success with links to Account and Data Overview

### Data Overview
- New page at `/account/data` showing what data FIO has for the logged-in user
- Company card with name, code, and faction
- Category grid with tick/cross indicators and last-updated timestamps for 9 data types
- Empty state directs users to Setup

### Bug fixes
- Fixed auth token race condition on page refresh — token now set synchronously during state initialization instead of in an effect, preventing 401-triggered session clearing
- Fixed RetrievalResponse extraction — API returns lowercase username keys, now uses case-insensitive lookup
- Fixed CX Orders field name (`CXOSs` not `CxOrders` in AllData response)

### Navigation
- Added "My Data" to sidebar and Account quick links
- Added Setup mention on registration page
- Setup is a public route (no sidebar) for pre-login accessibility

## 0.3.1 — 2026-04-02

### Registration page
- Updated password warning to match official FIO wording
- Missing-params page now explains the extension flow with numbered steps
- Added direct links to Chrome Web Store and Firefox Add-ons for the FIO extension

## 0.3.0 — 2026-04-02

### Phase 3: Account settings, webhooks, and admin panel
- Discord name inline editing on Account overview page
- Account Settings page: anonymous data collection preference, selective game data reset (10 categories), account deletion with two-step password confirmation
- Webhooks page: display/set/delete webhook URL, handles 406 for non-bot accounts (save currently disabled pending API verification)
- Admin panel: check user, create account (with admin/bot flags), delete account, 7 data management clear operations with type-to-confirm safety
- AdminRoute guard component with dev override for UI testing
- ConfirmDialog now supports `requireConfirmText` for type-to-confirm destructive actions

### Cleanup
- Removed mock data layer (`mock.ts`) and all `USE_MOCK` references
- API modules (`auth.ts`, `apikeys.ts`, `permissions.ts`, `groups.ts`) simplified to direct API calls

### Routing & navigation
- New sidebar links: Settings, Webhooks, Admin Panel (admin-only)
- New routes: `/account/settings`, `/account/webhooks`, `/admin`
- UserAutocomplete added to admin check user and delete account fields

## 0.2.0 — 2026-04-02

### Live API integration
- Reconnected all API calls to `api.fnar.net` (disabled mock mode)
- Restored auth guard on protected routes
- Added Vite dev proxy (`/api` → `api.fnar.net`) to avoid CORS issues in development
- Dev server uses proxy; production builds hit the API directly

### Registration
- Registration now reads `UserName` and `RegistrationGuid` from URL query params (generated by the FIO extension)
- Username field is pre-filled and read-only; registration GUID is hidden
- Missing query params show a message directing users to the FIO extension

### Permissions
- Fixed permission field names to match actual API response (unprefixed: `Information` not `ShipInformation`, etc.)
- Fixed permission counts, checkbox population, and save persistence
- Edit permissions now opens as a modal dialog instead of inline expand
- Added Select All / Clear All buttons to grant and edit permission dialogs
- Granted to Me tab now shows compact cards with a View button instead of inline permission grids

### Groups
- Added Select All / Clear All buttons to group creation permissions
- Group name input validates 3–16 characters; API validation errors are parsed into readable messages
- Invite, Kick, and Promote actions use consistent inline pop-down panels with smooth animation (no modals)
- Invite input upgraded to multi-select tag input (select multiple users, add once)
- Self-invite prevented on group creation; already-invited users filtered from dropdown
- Sidebar shows invite count badge on Invitations nav item, updates on accept/reject

### Login
- Remember Me checkbox now persists username to localStorage

### UI fixes
- Fixed `input[type="number"]` not picking up theme styles
- Fixed `autoComplete` on API key application name field triggering password manager
- Replaced all bare `\u` unicode escapes in JSX with actual characters
- Removed inaccurate "won't be shown again" warning from API key creation
- Added `btn-sm` utility class to global styles
- Added keyboard navigation (arrow keys, Enter, Tab, Escape) to UserAutocomplete dropdown

## 0.1.0 — 2026-04-01

Initial implementation: Phase 1 (auth, account, API keys) and Phase 2 (permissions, groups) with mock data layer, three-theme system, and responsive layout.
